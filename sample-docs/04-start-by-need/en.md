# Start by Need

> **Suggested CTA placement:** Bottom of Quick Start; next to "Get started" on homepage; post-signup onboarding page

Not sure where to begin? Find the scenario that matches yours and jump right in.

> **First step for all deployment scenarios:** You need a dedicated server. You can [buy one from Zeabur](https://zeabur.com/docs/en-US/dedicated-server) or [register your own machine](https://zeabur.com/docs/en-US/dedicated-server/register). Once you have a server, proceed based on your needs below.

---

## "I already have my own server and want Zeabur to manage it"

If you've rented a machine elsewhere (AWS, GCP, Hetzner, etc.), you can register it with Zeabur and use Zeabur's interface to manage deployments, monitoring, and scaling — no need to handle Docker and CI/CD yourself.

**Start here:** [Dedicated Server Overview](https://zeabur.com/docs/en-US/dedicated-server) → [Register Server](https://zeabur.com/docs/en-US/dedicated-server/register)

---

## "I want to buy a server through Zeabur"

Zeabur offers dedicated server plans with fully isolated resources. You can manage upgrades, restarts, and monitoring directly from the Zeabur dashboard.

**Start here:** [Dedicated Server Overview](https://zeabur.com/docs/en-US/dedicated-server) → [Operate Server](https://zeabur.com/docs/en-US/dedicated-server/operate)
**Pricing:** [Plans & Pricing](https://zeabur.com/docs/en-US/billing/pricing)

---

## "I just want to try it out — I don't have a project yet"

Start with a template. Zeabur has hundreds of pre-configured templates — blogs, automation tools, databases — ready to deploy in a few clicks.

**Start here:** [Deploy from Template](https://zeabur.com/docs/en-US/template/deploy-template)
**Then:** [Quick Start](https://zeabur.com/docs/en-US/get-started)

---

## "I have a project and want to deploy it"

Connect your GitHub repo to Zeabur. It auto-detects your framework and language, builds, and deploys. Every future push auto-updates.

If your project was generated by an AI tool (v0, Replit, Lovable, Bolt, etc.), we have dedicated import guides.

**Start here:** [Create Service](https://zeabur.com/docs/en-US/deploy/create-service) → choose GitHub
**Framework setup:** [Language & Framework Guides](https://zeabur.com/docs/en-US/guides/nodejs) (Node.js, Python, Go, etc.)
**From AI tools:** [Import Guides](https://zeabur.com/docs/en-US/tutorials) (v0, Replit, Lovable…)

---

## "I want to migrate from another platform"

If you're currently on Heroku, Railway, Vercel, or another platform, Zeabur's deployment model is similar. Most projects can migrate directly.

**Start here:** [Migrate from Other Platforms](https://zeabur.com/docs/en-US/tutorials/migration) (🚧 Coming soon)
**Environment setup:** [Environment Variables](https://zeabur.com/docs/en-US/deploy/variables)

---

## "I want to use AI features"

Zeabur AI Hub gives you access to multiple AI models (OpenAI, Claude, etc.) through a single interface with credit-based billing — no need to get API keys from each provider separately.

**Start here:** [Zeabur AI Hub](https://zeabur.com/docs/en-US/ai-hub)

---

## "I want to set up email for my product or team"

Zeabur Email lets you send transactional emails, verification emails, and notifications using your own domain. It comes with a REST API and webhooks — ideal for products or teams that need branded email.

**Start here:** [Zeabur Email](https://zeabur.com/docs/en-US/email) (currently in beta)

---
