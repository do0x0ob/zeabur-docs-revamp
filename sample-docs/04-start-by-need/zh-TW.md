# 依需求開始

> **CTA 建議位置：** 快速入門底部、首頁立即開始旁邊、註冊完成後引導頁

不確定從哪裡開始？找到最接近你的情況，直接跳到對的地方。

> **所有部署情境的第一步：** 你需要一台專用伺服器。可以[在 Zeabur 購買](https://zeabur.com/docs/zh-TW/dedicated-server)或[註冊自己的機器](https://zeabur.com/docs/zh-TW/dedicated-server/register)。有了伺服器之後，再依你的需求往下走。

---

## 「我已經有自己的 Server，想用 Zeabur 管理」

如果你在外面（AWS、GCP、Hetzner 等）已經租了機器，可以把它註冊到 Zeabur，用 Zeabur 的介面來管理部署、監控和擴展，不用自己搞 Docker 和 CI/CD。

**開始：** [專用伺服器概述](https://zeabur.com/docs/zh-TW/dedicated-server) → [註冊伺服器](https://zeabur.com/docs/zh-TW/dedicated-server/register)

---

## 「我想直接在 Zeabur 買 Server」

Zeabur 提供專用伺服器方案，買了之後可以在上面建立專案和服務，資源完全獨立。從 Zeabur 介面就能管理機器的升級、重開、監控等操作。

**開始：** [專用伺服器概述](https://zeabur.com/docs/zh-TW/dedicated-server) → [維護伺服器](https://zeabur.com/docs/zh-TW/dedicated-server/operate)
**定價：** [方案與定價](https://zeabur.com/docs/zh-TW/billing/pricing)

---

## 「我想先試試看，還沒有自己的專案」

從模板開始最快。Zeabur 有上百個預設模板，涵蓋部落格、自動化工具、資料庫等，點幾下就能上線。

**開始：** [從模板部署](https://zeabur.com/docs/zh-TW/template/deploy-template)
**接著看：** [快速入門](https://zeabur.com/docs/zh-TW/get-started)

---

## 「我有一個專案，想部署上去」

把你的 GitHub repo 連上 Zeabur，它會自動偵測框架和語言，幫你建置和部署。之後每次 push 都自動更新。

如果你的專案是用 AI 工具產生的（v0、Replit、Lovable、Bolt 等），我們也有專門的導入指南。

**開始：** [建立服務](https://zeabur.com/docs/zh-TW/deploy/create-service) → 選 GitHub
**框架設定：** [語言與框架指南](https://zeabur.com/docs/zh-TW/guides/nodejs)（Node.js、Python、Go 等）
**從 AI 工具來的：** [導入指南](https://zeabur.com/docs/zh-TW/tutorials)（v0、Replit、Lovable…）

---

## 「我想從別的平台搬過來」

如果你目前用 Heroku、Railway、Vercel 或其他平台，Zeabur 的部署方式很相似，大部分專案可以直接搬。

**開始：** [從其他平台遷移](https://zeabur.com/docs/zh-TW/tutorials/migration)（🚧 建置中）
**環境設定：** [環境變數](https://zeabur.com/docs/zh-TW/deploy/variables)

---

## 「我想用 AI 功能」

Zeabur AI Hub 讓你透過單一介面存取多種 AI 模型（OpenAI、Claude 等），用點數計費，不用各平台分別申請 API key。

**開始：** [Zeabur AI Hub](https://zeabur.com/docs/zh-TW/ai-hub)

---

## 「我想為產品或團隊架設 Email 服務」

Zeabur Email 讓你用自己的域名發送交易信、驗證信、通知信，有 REST API 和 Webhook 可以串接，適合需要品牌信箱的產品或團隊。

**開始：** [Zeabur Email](https://zeabur.com/docs/zh-TW/email)（目前為 beta 測試中）
